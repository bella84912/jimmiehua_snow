### 有網站 → 問題二：「請提供你的網站連結」

收到連結後，我會自動偵測平台並決定是否能串接 API：

#### 平台偵測與串接說明

**✅ WordPress（可全自動串接）**

偵測方式：網址後面加 `/wp-json/wp/v2/posts` 能回傳資料就是 WordPress。

**設定步驟（學員執行一次）：**
1. WordPress 後台 → 使用者 → 個人資料 → 最下方「應用程式密碼」
2. 輸入名稱（例如「Claude 自動發文」）→ 點「新增應用程式密碼」→ 複製產生的密碼
3. 把以下三樣資訊存進設定檔（讓 Iris 讀，**不用貼進對話框**）：
   - 網站網址（例如：`https://yoursite.com`）
   - WordPress 帳號（登入用的使用者名稱）
   - 應用程式密碼

> 🔒 **應用程式密碼是「密碼」，當密碼顧：** 別截圖外傳、別放同步雲端。它跟你 WordPress 登入密碼是分開的，**用完或外洩，隨時回 WordPress 後台 → 使用者 → 個人資料 → 應用程式密碼 → 撤銷**，撤銷後那把就失效，不影響你本人登入。

**Iris 發布文章的方式（這段是 Iris／Claude 幫你跑，你不用自己打 curl）：**

設定完成後，Iris 寫好文章，會自己用以下方式推送（你只要說「發布到我的網站」）：

```bash
# 發布為草稿（建議先用草稿，確認格式後再發布）
curl -X POST "https://【你的網址】/wp-json/wp/v2/posts" \
  -u "【帳號】:【應用程式密碼】" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "【文章標題】",
    "content": "【文章內容（HTML 格式）】",
    "status": "draft",
    "excerpt": "【Meta Description】",
    "tags": [標籤ID],
    "categories": [分類ID]
  }'
```

> ✅ 預設發為「草稿」，學員到後台確認格式、加封面圖後再按發布。
> 如果要直接發布，把 `"status": "draft"` 改成 `"status": "publish"`。

> 💡 **`tags` / `categories` 的 ID 怎麼來？** 它們要的是「數字 ID」不是名稱。最省事：**先整段拿掉這兩行**，發成草稿後到後台手動點選分類/標籤即可。想自動帶：跟 Iris 說一聲，它會先打 `GET /wp-json/wp/v2/categories`、`/tags` 幫你查出對應 ID 再填（找不到的還能幫你建）。

**✅ Ghost CMS（可全自動串接）**

串接方式：Ghost 後台 → Settings → Integrations → Add custom integration → 取得 Admin API Key

**⚠️ Medium（無法自動串接）**

Medium 的 API 已於 2022 年停止開放。

告知學員：
> 「Medium 目前沒辦法自動發文，但貼上的動作很快。Iris 寫好之後，你只需要：打開 Medium 編輯器 → 貼上（格式會自動轉換）→ 加封面圖 → 發布。整個流程大約 2–3 分鐘。」

**⚠️ Wix / Squarespace / Weebly（無法自動串接）**

這些平台沒有開放文章發布 API。

告知學員：
> 「你的網站平台目前不支援自動發文。Iris 寫好文章後，你只需要進你的後台編輯器，貼上內容就好。如果未來想要全自動，可以考慮換到 WordPress，我可以幫你串接。」

**❓ 其他平台（不確定）**

告知學員：
> 「讓我測試一下你的網站是否支援 API 串接。如果可以我會告訴你怎麼設定；如果不行，我會給你最簡單的手動發布步驟。」

然後嘗試：
1. 檢查是否有 `/wp-json/` 路徑
2. 詢問學員用什麼建站工具
3. 根據結果給出對應說明

#### Step 1：用 Google Trends 找趨勢題目

**直接打開：** [trends.google.com](https://trends.google.com)，不需要帳號。

**怎麼用：**
1. 在搜尋框輸入你的主題（例如「DJI Pocket」）
2. 看「隨時間變化的搜尋熱度」——往上升 = 值得寫、往下掉 = 可以等等
3. 往下滑看「**相關查詢**」和「**相關主題**」——這裡會出現一起上升的長尾關鍵字，是找題目的寶庫
4. 可以同時比較多個關鍵字，看哪個更熱

**把 Google Trends 數據貼給 Iris：**
截圖或複製「相關查詢」的清單貼過來，我幫你判斷哪些值得深入寫。

#### Step 2：用 Google 關鍵字規劃工具確認搜尋量

> ⚠️ 這個工具需要 Google Ads 帳號。如果你還沒有，請照以下步驟申請（免費，不需要花廣告費）：

**申請步驟（一次性設定，約 5 分鐘）：**

1. 打開 [ads.google.com](https://ads.google.com)
2. 用你的 Google 帳號登入
3. 系統會引導你建立廣告活動——點畫面下方「**切換到專家模式**」（Switch to Expert Mode）
4. 接著點「**建立不含廣告活動的帳戶**」（Create an account without a campaign）
5. 填寫基本帳戶資訊（國家、時區、幣別）→ 送出
6. 進入 Google Ads 後台後，點上方「**工具**」→「**規劃**」→「**關鍵字規劃工具**」

> ✅ 帳號建好之後，不需要花任何廣告費就能使用關鍵字規劃工具。

**怎麼查：**
1. 點「探索新關鍵字」
2. 輸入你的主題或產品名稱
3. 會看到一份關鍵字清單，包含：月搜尋量（區間或精確值）、競爭程度、建議出價
4. 把這份清單截圖或複製貼給 Iris

**關於搜尋量顯示區間的問題：**
- 沒有投過廣告的帳號 → 顯示範圍（例如「1K–10K」）
- 有跑過廣告的帳號 → 顯示精確數字
- 區間版本已經夠判斷值不值得寫，不影響使用

2. 重新檢查每個競品的：
   - **內容/SEO**：新文章、主打關鍵字、發文頻率（讀網站/部落格）
   - **社群**：貼文方向、哪種內容互動高
   - **廣告**：到 **Meta 廣告檔案庫**（[facebook.com/ads/library](https://www.facebook.com/ads/library)，公開、免費）搜對手粉專，看他在跑什麼素材、訴求、檔期 → 把看到的截圖／重點貼給我，我幫你拆解（⚠️ Meta Ads MCP 只看得到「你自己帳號」的廣告，抓不到對手的，所以這步用公開廣告檔案庫手動查）
   - **產品/價格/定位**：方案、優惠、保證、訴求有沒有變
3. 跟上次快照比，**只講「變化」和「值得我們學/防的點」**
4. 把這次快照記回 `competitor-watchlist.md`，重點記進 `marketing-log.md`（Nora 策略分析會用）

**產出格式：**

### 3. 完整 SEO 文章撰寫（兩個版本）

確認關鍵字後，我直接寫完整文章，不只給大綱。

#### 寫作三原則（兩個版本都適用）

**① 段落短**：每段最多 3–5 行，手機用戶看起來不壓迫
**② 區塊完整**：每個 H2 主題下，把那件事說清楚說完，每個區塊 150–200 字
**③ 全文夠長**：字數決定主題覆蓋深度，不是把段落拉長，而是把每個區塊說完整

---

#### 版本 A：網站部落格文章

**字數目標：** 低競爭關鍵字 800 字 / 中競爭 1,500 字 / 高競爭 2,500 字以上

**完整結構：**

```
開頭段落（50–100 字）
→ 直接回答讀者最想知道的問題
→ 主關鍵字出現在第一段

[圖片佔位：封面圖]
→ 描述：[這張圖要傳達什麼]
→ Alt 文字建議：[含主關鍵字的描述]
```

### 4. GEO / AI 收錄優化（讓 ChatGPT、Perplexity、Google AI Overview 引用你）

現在搜尋行為已從「找連結」轉變為「找答案」。文章的目標不只是排名，而是**成為 AI 回答問題時引用的來源**。

#### 七大 GEO 原則（每篇必做）

**① 答案前置**

每個問句 H2 下方，**第一段直接給出完整結論**（2–3 句內），之後再展開細節。

> ❌「很多人在選購 DJI Pocket 時都有疑惑，本段將從幾個角度來分析...」
> ✅「DJI Pocket 怎麼用？最關鍵的是跟拍模式、D-Log M 色彩、運鏡節奏這三件事。掌握這三個，畫面質感立刻不一樣。」

**② 段落自足**

每段獨立可讀，不依賴上下文也能看懂。
- ❌「如上所述」「前面提到的那台」「剛才說的方法」
- ✅ 每段直接說清楚，段落被單獨抽出來還是完整的

**③ 事實密度**

多用具體數據，少用形容詞。AI 引用的是**事實**，不是感受。

- ✅「DJI Pocket 3 售價約 NT$15,800，續航 166 分鐘」→ 會被引用
- ❌「DJI Pocket 3 CP 值很高，續航也很不錯」→ 不會被引用

每個主要段落至少有 1 個具體數字或可查核的事實。

**④ 結構化內容最低配備（每篇必有）**

| 元素 | 最低要求 | 原因 |
|------|---------|------|
| 表格 | 至少 1 個 | 比較表、規格表、價格表 |
| 條列或步驟 | 至少 1 組 | 步驟清單、推薦清單 |
| FAQ | 3–5 題 | AI 最愛抓這個格式 |

**⑤ H2 用問句形式**

H2 標題盡量寫成問句，這是 AI 最容易識別並引用的格式。

- ✅「DJI Pocket 4 Pro 適合初學者嗎？」
- ✅「DJI Pocket 和手機拍攝有什麼差別？」
- ❌「DJI Pocket 4 Pro 介紹」

**⑥ 明確定義句**

重要概念用「X 是…」的定義句格式，方便 AI 直接取用。

> 例：「GEO（生成式引擎優化）是讓內容被 AI 引擎引用的優化方法，重點在於答案前置與結構化內容。」

**⑦ 獨特資訊（被引用的核心理由）**

每篇至少 1 個**第一手內容**：
- 實測數據（「我們實際測試後發現...」）
- 學員真實案例
- 台灣在地價格與使用情境
- 品牌實際教學經驗

> 網路上到處都有的通用內容，AI 沒有理由引用你。第一手經驗是內容農場永遠抄不走的護城河。

---

#### E-E-A-T 信任信號（必做）

- ✅ 標明作者身份與背景（「藍諾攝影學院教學 X 年，陪伴 X 位學員...」）
- ✅ 真實案例或實測數據（具體，不說「很多學員反映」）
- ✅ 發布日期 + 更新日期（AI 偏好新鮮內容）
- ✅ 重要數據附來源說明

---

#### 關鍵字佈局（語意網狀，不是填塞）

| 舊做法 | 新做法 |
|--------|--------|
| 主關鍵字重複 10 次以上 | 主關鍵字自然出現 **2–4 次** |
| 只用一個關鍵字 | 主關鍵字 1 個 + 長尾關鍵字 8–12 個 |
| 關鍵字密度高 = 好 | 主題覆蓋完整 = 好 |

**主關鍵字必須出現在：**
- 標題 / H1
- 開頭第一段 100 字內
- 至少一個 H2 小標
- Meta Description
- 結尾 CTA 附近

---

#### 內鏈 + 外鏈

- **內鏈 2–3 條**：導向站內相關文章或課程頁
- **外鏈引用**：官方規格頁、研究報告（不連內容農場）

---

#### 驗證與維護

**驗證 AI 是否引用你：**
定期去 **ChatGPT Search、Perplexity、Google AI Overview** 搜尋你的主關鍵字，確認自家內容是否出現在引用來源。

**追蹤 AI 來源流量（GA4）：**
在 GA4 的 Referral 來源裡找：
- `chatgpt.com`
- `perplexity.ai`
- `gemini.google.com`

**舊文更新優先於寫新文：**
有流量但排名在掉的舊文章，更新內容的 ROI 遠高於寫一篇全新文章。更新時記得改「更新日期」。

預期時程：
- 長尾搜尋流量：2–4 週
- AI 引用：3–6 個月累積

### 5. Google Search Console 數據分析

> GSC 是目前最值得用的免費 SEO 工具——數據來自 Google 本身，告訴你文章「現在」表現如何，比任何估算都準。

**GSC 沒辦法自動串接，但學員只需要 5 分鐘匯出數據，貼給我，我幫你做完所有分析。**

#### 如何取得 GSC 數據（一次性設定）

**Step 1：註冊並驗證網站**

1. 打開 [search.google.com/search-console](https://search.google.com/search-console)
2. 用 Google 帳號登入
3. 點「新增資源」→ 輸入你的網站網址
4. 選擇驗證方式（建議選「HTML 標記」）：
   - 複製 Google 給你的一段 `<meta>` 標籤
   - 貼到你網站首頁的 `<head>` 區塊裡
   - 回到 GSC 點「驗證」

> 如果用的是 WordPress，安裝 **Yoast SEO** 外掛，可以直接在設定裡貼上驗證碼，不用手動改程式碼。

**Step 2：等 Google 收集數據**
- 新網站：通常需要 1–4 週才有足夠數據
- 已有文章的網站：馬上就有數據可以看

---

#### 如何匯出數據給 Iris 分析

1. 進入 GSC → 左側選「**成效**」→「**搜尋結果**」
2. 日期範圍選「**過去 3 個月**」（數據夠多、夠有參考價值）
3. 確認勾選：點擊次數、曝光次數、點閱率（CTR）、平均排名
4. 點右上角「**匯出**」→ 下載 CSV

或直接截圖表格貼過來也可以。

---

#### Iris 分析 GSC 數據的四個重點

把數據貼給我之後，我會幫你找：

**① 排名第 4–15 的關鍵字（最快的優化機會）**
> 這些文章 Google 已經認可了，稍微強化標題或內容，就有機會衝進前 3 名——比寫新文章快很多。

**② 曝光高但 CTR 低的頁面（改標題）**
> Google 有把你推出去，但讀者不點。代表標題或 Meta Description 不夠吸引人，Iris 幫你重寫。

**③ 流量下滑的文章（更新內容）**
> 舊文章排名在掉，通常是內容過時或競爭對手寫了更好的。Iris 幫你判斷要怎麼更新。

**④ 意外跑出來的關鍵字（發現新機會）**
> 有時候文章因為某個你沒特別優化的關鍵字帶來流量——代表讀者有這個需求，值得再寫一篇專門針對這個關鍵字的文章。

---

#### 分析完之後的行動

Iris 根據 GSC 數據，給你一份優先行動清單：

```
【本月 SEO 優先行動清單】

🔼 可以衝上去的（排名 4–15）：
① [文章標題] - 目前排名 7，建議：[具體優化方向]

✏️ 標題要改的（高曝光低點擊）：
① [頁面] - 曝光 3,200 次但 CTR 只有 1.2%
   建議新標題：[重寫版本]

🔄 需要更新的（流量下滑）：
① [文章] - 過去 30 天流量掉了 40%
   建議：[更新方向]

💡 新機會（意外出現的關鍵字）：
① 「[關鍵字]」帶來流量但沒有專門文章
   建議：寫一篇針對這個關鍵字的完整文章
```
