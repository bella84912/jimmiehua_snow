# Workflow：競品文章拆解

> 丟競品 URL，Iris 產出對方的 SEO 結構與超越策略。

---

## 觸發條件

- 「拆這篇文章」+ URL
- 「分析競品 SEO」
- 「為什麼這篇排名好」

---

## 執行步驟

### Step 1：拿到 URL + 用戶想超越的關鍵字

問清楚：
- 競品文章 URL
- 你想針對什麼關鍵字超越他？
- 你目前自己寫的版本（如果有）

### Step 2：分析競品文章

如果有 WebFetch / Firecrawl 工具，直接抓內容分析。沒有的話，請用戶貼上完整內容。

> 🚫 **鐵則：沒有「真的拿到全文」，就不准輸出精確數字。**
> 總字數、主關鍵字出現次數、內鏈數、圖片數這類「用數的」指標，**只有在我實際抓到（或用戶貼上）整篇文章時才填**。
> 如果只有 URL、抓取失敗、或用戶沒貼全文——**這些欄位一律標「需抓取全文確認」，或改用相對描述（如「字數偏多／內鏈偏少」），絕不憑印象填一個精確數字。**
> 編一個「總字數 2,400」出來看似專業，其實是瞎猜，會誤導用戶的競爭判斷。

分析以下 6 個面向：

#### 1. 結構分析
- 文章總字數
- H1 / H2 / H3 層級
- 段落數量與長度
- 是否有目錄

#### 2. 關鍵字布局
- 主關鍵字出現次數與位置（標題 / 第一段 / H2）
- 相關長尾關鍵字
- LSI 關鍵字（語意相關詞）

#### 3. 標題公式
- 主標題用什麼模板（數字型 / 問題型 / 怎麼做型）
- 副標題的節奏
- 標題是否有情緒字

#### 4. 內鏈設計
- 內鏈數量
- 連到哪些頁面（產品頁 / 其他文章 / FAQ）
- 錨點文字策略

#### 5. 外鏈與權威
- 引用了哪些外部來源
- 是否有數據、研究、官方資料

#### 6. 視覺與多媒體
- 圖片數量與類型
- 是否有影片
- 表格 / 圖表使用

### Step 3：產出拆解報告

```markdown
# 競品拆解｜[文章標題]

## 一句話評語
[這篇為什麼會排名好的核心原因]

## SEO 結構數據
（⚠️ 只在「有抓到全文」時填精確數字；沒抓到就填「需抓全文確認」或相對描述）
| 指標 | 數值 |
|------|------|
| 總字數 | XXXX |
| H2 數量 | X |
| H3 數量 | X |
| 內鏈數 | X |
| 圖片數 | X |

## 關鍵字策略
- **主關鍵字**：[詞 + 出現次數]
- **次要關鍵字**：[3-5 個]
- **長尾關鍵字**：[3-5 個]

## 文章節奏分析
1. **開場**（前 200 字）：[用什麼方法抓住讀者]
2. **核心論述**（H2 段落）：[每個 H2 在解決什麼問題]
3. **行動引導**：[CTA 設計]

## 強項（我們要學的）
1. [優勢點 1]
2. [優勢點 2]
3. [優勢點 3]

## 弱點（我們可以打敗的地方）
1. [弱點 1，例：沒提到某個關鍵 use case]
2. [弱點 2，例：見證不夠]
3. [弱點 3，例：CTA 太弱]
```

### Step 4：產出超越策略

```markdown
## 你的超越策略

### 字數策略
建議目標：[X 字]（比競品多 30-50%）

### 結構升級
- 多加 X 個 H2，補上競品沒寫的：[具體主題]
- 加入競品沒有的元素：[例：實際操作截圖 / 影片 / 比較表]

### 關鍵字深化
- 補上競品漏掉的長尾：[3-5 個關鍵字]
- 增加 FAQ 段落（吃 Google 的 People Also Ask）

### 差異化內容
- 你的獨家觀點是 [XXX]，這是競品沒有的
- 加入你的數據 / 案例 / 學員見證

### 內鏈策略
- 從 [你的 LP] 連過來
- 連向 [你的銷售頁]
```
