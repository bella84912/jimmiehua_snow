# Workflow：SEO 大綱 + Meta 標題

> 丟關鍵字進來，Iris 產出 H2 / H3 大綱、目標字數、Meta title / description。

---

## 觸發條件

- 「給我 [關鍵字] 的 SEO 大綱」
- 「寫 Meta 標題」
- 「我要寫一篇 SEO 文章，幫我規劃結構」

---

## 執行步驟

### Step 1：確認關鍵字 + 寫作目標

問用戶：
1. 主關鍵字是什麼？
2. 這篇文章要達成什麼？（收名單 / 導向產品頁 / 純漲流量）
3. 預計多少字？（沒概念的話我給建議）

### Step 2：研究搜尋意圖

針對這個關鍵字，問自己：
- 搜這個詞的人最想知道什麼？
- 他們是想買、想學、還是想比較？
- 他們的搜尋背景是什麼？（剛入門 vs 有經驗）

從 `_brand/icp.md` 找對應的 persona，反推搜尋情境。

### Step 3：產出完整大綱

```markdown
# SEO 大綱｜[關鍵字]

## 文章基本資訊
- **主關鍵字**：[詞]
- **次要關鍵字**：[3-5 個長尾]
- **目標字數**：[1500-3000 字依競爭度]
- **搜尋意圖**：[資訊型 / 商業型 / 交易型]
- **適合 CTA**：[收名單 / 導 LP / 導 SP]

---

## Meta 標題

### Title（55 字內）
**版本 A**：[強調好處型，X 字元]
**版本 B**：[強調數字型，X 字元]
**版本 C**：[強調問題型，X 字元]

→ 建議使用：[A / B / C，附原因]

### Description（155 字內）
[Meta description 文字，含主關鍵字，X 字元]

---

## 文章大綱（H1 → H2 → H3）

### H1（主標題）
[包含主關鍵字 + 情緒字 / 數字]

### 開場（前 200 字）
- Hook：[第一句話策略]
- 同理用戶痛點
- 預告文章會解決什麼

### H2 #1：[次要關鍵字 1]
- H3：[子主題 1.1]
- H3：[子主題 1.2]
- 重點：[這段要傳遞的核心訊息]

### H2 #2：[次要關鍵字 2]
- H3：[子主題 2.1]
- H3：[子主題 2.2]
- 重點：[核心訊息]

### H2 #3：[次要關鍵字 3]
- H3：[子主題 3.1]
- H3：[子主題 3.2]
- 重點：[核心訊息]

### H2 #4：常見問題 FAQ（吃 People Also Ask）
- Q1：[最常被搜的問題]
- Q2：[第二常見]
- Q3：[第三常見]

### H2 #5：結論 + CTA
- 重申文章重點
- 引導到 [LP / SP / 訂閱]
- CTA 文字建議

---

## 內鏈規劃
- 連向 [你的 LP]：錨點文字 [建議用詞]
- 連向 [相關文章 A]：錨點文字 [建議用詞]
- 連向 [產品頁]：錨點文字 [建議用詞]

## 外鏈建議
- 引用 [權威來源]，例：[Statista / 政府網站 / 學術研究]
- 不要連到競品

## 多媒體建議
- 主視覺 1 張（Hero）
- 內文圖 X 張（每個 H2 至少 1 張）
- 表格 / 對比圖 X 個
- 影片嵌入（如果有）

---

## 寫作節奏建議
1. **每 200 字一個 H3 或視覺元素**（避免讀者疲勞）
2. **首段 + 末段必含主關鍵字**
3. **H2 之間用「過渡句」連接**，不要硬切
4. **第一個 CTA 出現位置**：第 1/3 文章長度處（不要等到最後才喊）
```
