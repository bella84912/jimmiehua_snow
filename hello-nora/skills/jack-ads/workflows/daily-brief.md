> 📋 **注意**：本文件中所有 `ROAS`、`CTR`、`CVR` 等具體數字（如 3.2、0.5%）**皆為格式示範、非真實數據**。實際報告一律以你帳號拉到的真實數字為準；沒有真實數據時不編造。

## 免費版執行流程

### Step 1：請用戶提供數據

問用戶把昨天的廣告數據貼進來，至少要這 6 個：

1. **曝光（Impressions）**
2. **點擊率 CTR**
3. **每次點擊成本 CPC**
4. **LP 瀏覽率**（點擊 → 進 LP 的比例）
5. **購買轉換率 CVR**
6. **廣告投資回報率 ROAS**

如果有多組廣告，把每組分開貼。

### Step 2：對每組廣告做健康度評分

```markdown
## 廣告組 #1：[名稱]

| 指標 | 昨日數字 | 健康度 | 標準 |
|------|---------|--------|------|
| CTR | X% | 🟢/🟡/🔴 | 一般 1.5%+ |
| CPC | NT$X | 🟢/🟡/🔴 | 依產品定價 |
| CVR | X% | 🟢/🟡/🔴 | 一般 2%+ |
| ROAS | X.X | 🟢/🟡/🔴 | 至少 1.5（盈虧平衡）|
```

健康度判讀：
- 🟢 高於行業標準
- 🟡 接近標準，需注意
- 🔴 明顯低於標準

### Step 3：給明確建議

對每組廣告，給 1 個明確行動：

```markdown
### 廣告組 #1：建議「加碼」🟢
- ROAS 3.2 表現極好
- 建議：每日預算 +30%（從 1000 加到 1300）
- 注意：別一次加太多，避免演算法重新學習

### 廣告組 #2：建議「暫停」🔴
- CTR 0.5% 太低，受眾沒興趣
- 建議：今天就關掉，下次換素材測試
- 啟動 hook-writing workflow 寫新的廣告 hook

### 廣告組 #3：建議「換素材」🟡
- CTR 1.2% 還行，但 CVR 0.5% 太差
- 問題：點擊進去後沒有轉換
- 建議：受眾沒問題，是 LP 或 hook 跟產品不夠對齊
- 下一步：Leon 看一下 LP，或 Jack 換 hook 角度
```

### Step 4：整體判斷

```markdown
## 整體一句話

**今日重點**：[1 件最重要的事]
**今日行動**：
1. [優先級 1 的動作]
2. [優先級 2 的動作]
3. [優先級 3 的動作]

**這週累計趨勢**：
- 上週 ROAS：X.X
- 本週至今：X.X
- 趨勢：[上升 / 持平 / 下降]
```

## Claude 付費版版執行流程

> ⚠️ **先有心理準備**：下面這些「自動拉、每天自動推、自動調預算」**需要 Meta Ads MCP 接得上**，而官方免費 MCP 目前全球分批開放、**台灣帳號可能還沒輪到**；自動推播與自動調預算還要另外接排程／規則。**台灣現階段多數人是走「手動貼數據、我來分析」**，接得上自動版再走下面流程，別預設一定能用。

### Step 1：自動拉數據

透過 Meta Ads MCP 自動取得昨天所有廣告的：
- 曝光 / 點擊 / CTR
- 花費 / CPC
- 結帳 / CVR
- 業績 / ROAS

### Step 2：自動分析 + 推播

每天早上 9:00 自動：
1. 拉數據
2. 跑健康度評分
3. 產出建議
4. 推到用戶的 Claude / Email / Slack

### Step 3：高自動化（最強版）

設定規則：
- ROAS > 2.5 自動加 20% 預算
- CTR < 0.5% 自動暫停廣告組
- 連續 3 天 CVR < 1% 觸發「需要 Leon 看 LP」的警報
