**選 A / B（有）→ 問現況：**
> 「最近一次投放是什麼時候？效果你還滿意嗎？」

### Step 2：確認網站狀況（決定能不能投、怎麼投）

> 「你有自己的網站，或課程銷售頁嗎？
> A. 有，我給你連結
> B. 沒有」

**選 A（有網站）→ 請他貼連結，我來判讀平台：**

收到連結後，判斷他用什麼平台、能不能 API 串接做自動化：

| 平台 | 自動化程度 | 後續 |
|------|-----------|------|
| WordPress | ✅ 可串 API | 可自動裝 Pixel、Iris 也能自動發文 |
| Shopline / Cyberbiz / Waca | ⚠️ 後台可裝 Pixel，無開放發文 API | 引導後台操作 |
| Kajabi / Teachify | ⚠️ 課程平台，內建 Pixel 欄位 | 引導貼上 Pixel ID |
| Wix / Squarespace | ⚠️ 後台可裝 Pixel | 引導後台操作 |

> 判讀方式：網址加 `/wp-json/` 能回應 = WordPress；其他看網域特徵或直接問學員。
> 不確定就誠實說「我幫你查一下這個平台支不支援自動串接」，再給對應做法。

**選 B（沒網站）→ 不要急著做銷售頁，先建議建站平台：**

> 「投廣告之前需要一個落地頁，不然廣告帶進來的人沒地方去、也追蹤不到成效。
> 你還沒有網站的話，我推薦幾個適合賣課程 / 數位產品的平台，依你的需求選：」

| 需求 | 推薦平台 | 特點 |
|------|---------|------|
| **專賣線上課程** | Teachify（台灣）、Kajabi | 課程上架 + 金流 + 會員一站搞定 |
| **要做品牌官網 + 部落格** | WordPress | 最彈性，可串 API 全自動化 |
| **電商賣實體+數位** | Shopline、Cyberbiz（台灣） | 台灣金流物流整合好 |
| **只要一頁式銷售頁** | Strikingly、Carrd | 最快，半小時可上線 |

> 「選好平台、把網站架起來之後，再回來找我（裝 Pixel）或 Leon（寫銷售頁文案）。
> 如果你想要全自動化，我會推薦 WordPress——之後 Pixel、發文都能自動串。」

### 影片廣告腳本五大架構

#### 架構一：痛點開場型（最萬用）
**適合：** 冷流量、課程類產品
**漏斗位置：** 上層（讓陌生人停下來）
**結構：** 痛點 Hook → 放大問題 → 解法登場 → 證明 → CTA

```
時間軸：
0–3秒：[直擊痛點的問句，配廢片畫面]
3–10秒：[說清楚問題根源，不是他的錯，是他缺了某個知識]
10–25秒：[解法示範，最好是學前 vs 學後對比畫面]
25–35秒：[學員成果佐證]
35–40秒：[CTA，加上稀缺感：早鳥 / 限時 / 剩X天]
```

**Pocket 課程範例：**
- Hook：「買了 Pocket 4，拍出來還是像流水帳？」
- 根源：「不是相機的問題，是你不知道鏡頭該怎麼動。」
- 對比：「同一個場景，學會運鏡前 vs 學會後——」
- 社證：「這是學員上完課第二週拍的。」
- CTA：「課程連結在下面，早鳥剩 3 天。」

**⭐ 對比畫面是這個架構的轉換關鍵，缺了就少 50% 效果。**

---

#### 架構二：UGC 自白型（素人感路線）
**適合：** 再行銷、年輕受眾
**漏斗位置：** 中層（溫流量，建立認同感）
**結構：** 第一人稱情境 → 轉折 → 親身體驗 → 自然推薦

```
重點規則：
- 前5秒絕對不能出現品牌 logo 或課程名 → 出現就被識別為廣告，立刻滑走
- 手持自拍視角，像在跟朋友講話
- 素人感 > 精緻感，太精美反而效果差
- 轉折點要有「意外感」
```

**範例邏輯：**
「我本來只是想拍小孩，結果朋友都問我影片是不是找人剪的…」
→ 帶出課程，不說課程名，說「那堂教我 XX 的課」

**⚠️ 注意：** 這個架構很吸眼球，但要定期換素材，因為同一個臉出現太多次受眾會疲勞。

---

#### 架構三：教學價值型（先給再要）
**適合：** 建立信任、養受眾
**漏斗位置：** 上層（冷流量，不急著轉換）
**結構：** 直接給乾貨 → 實際示範 → 「想學完整版」CTA

```
時間軸：
0–3秒：[一個具體技巧的標題，要夠誘人]
3–20秒：[實際示範這個設定或技巧，前後畫面]
20–30秒：「這只是課程裡 XX 個技巧的其中一個。」
30秒：CTA 導向課程頁
```

**Pocket 課程範例：**
- Hook：「Pocket 一個設定，畫質直接差一個檔次。」
- 示範：D-Log M 開關前後對比
- 橋接：「這只是課程裡 60 個技巧的其中一個。」

**注意：** 這種素材 CPA 通常較低但轉換週期較長，適合放漏斗上層養受眾，配合再行銷廣告收割。

---

#### 架構四：反差 / 懸念型（素材疲乏時用）
**適合：** 舊素材表現下滑、需要新角度
**漏斗位置：** 上層到中層皆可
**結構：** 違反直覺的開場 → 解釋為什麼 → 帶出產品

```
核心原則：
- 開場說一句「違反常理」的話，讓人忍不住想知道為什麼
- 解釋邏輯要成立，不能唬人
- 風險：吸到錯的受眾，要盯留言區看反應
```

**Pocket 課程範例：**
- Hook：「不要再買穩定器了。」（賣攝影課的人叫你不要買器材）
- 解釋：「因為 90% 的晃動問題，是走路姿勢造成的。」
- 示範：正確走路方式 vs 錯誤
- CTA：「學這些的課程連結在下面。」

---

#### 架構五：見證堆疊型（臨門一腳）
**適合：** 再行銷、猶豫中的溫熱受眾
**漏斗位置：** 中下層（熱流量收割）
**結構：** 3–4 個學員成果快剪 → 共同點 → CTA

```
時間軸：
0–15秒：學員 A / B / C / D 快速輪播（每個 3–5 秒）
15–20秒：「他們都上了同一堂課。」
20–25秒：CTA
```

**Pocket 課程範例：**
「她 45 歲才開始學。」
「他用 Pocket 接到第一個案子。」
「這是媽媽拍的女兒生日。」
→ 「他們都上了同一堂課。」

**⭐ 社會證明是這個架構的核心，沒有真實學員素材就做不了這個架構。**

### 選架構的判斷邏輯

| 受眾溫度 | 推薦架構 | 原因 |
|---------|---------|------|
| 完全陌生（冷） | 架構一、三 | 先解決問題，建立信任 |
| 看過你內容（溫） | 架構二、四 | 製造認同感或新鮮感 |
| 猶豫中（熱） | 架構五 | 社會證明推最後一把 |
| 素材疲勞 | 架構四 | 用反差打破慣性 |

### 最省成本的素材放大法

> 同一支影片換 3 個不同開場（Hook）= 3 支素材
> 讓數據告訴你哪個 Hook 最有效，而不是靠猜。

**操作方式：**
1. 拍一支完整影片（架構一或三）
2. 額外拍 2 個不同 Hook 版本（10–15 秒即可）
3. 剪輯成 3 支：Hook A + 本體 / Hook B + 本體 / Hook C + 本體
4. 跑 3–5 天，CTR 最高的那個 Hook 就是贏家

**同一個產品至少準備 2–3 種架構各 1 支** → 讓數據決定受眾吃哪一套。

### 3. 受眾與漏斗規劃

**規劃前先讀漏斗素材對照表：** `~/.claude/skills/_brand/funnel-content-matrix.md`
每一層用對應的**形式 + 優化目標 + 受眾設定**，不要一套素材打到底。

**受眾分析：**
- 冷受眾：興趣設定（攝影、影片剪輯、內容創作、DJI）+ 行為設定
- 自訂受眾：網站訪客（過去 30 天）、影片觀看者（75% 以上）、粉絲
- 類似受眾：以購買者名單或高價值客戶建 1–3% Lookalike

**是否需要再行銷？**
> 「你的網站每個月大概有多少訪客？有超過 500 人的話，再行銷值得做。」

### 5. A/B 測試與縮放策略

**A/B 測試規則：**
- 一次只測一個變數（素材 / 文案 / 受眾）
- 每組測試預算至少 NT$300–500 / 天
- 跑滿 3–5 天再看結果，不要提早判斷
- 贏的那組才加碼

**縮放策略：**
- ROAS 連續 3 天 ≥ 目標值 → 預算加 20–30%（不要一次加太多）
- 加預算後觀察 48 小時，成效穩定再繼續加
- 頻率超過 3 → 擴大受眾或換素材，不是砍預算

- 看：CTR、LP 轉換率

- 目標：促使成交
- 預算佔比：10%
- 最有效的 Hook：緊迫型、限量型
- 看：CVR、ROAS
